# features/observationBuild

観測テーマと生成モードを選び、初期生成に必要な前提を決めてキャリアを観測する画面です。

- `ObservationBuildScreen.tsx` テーマ選択、ビルド設定、新弟子札プレビュー、選定ログ、cost 表示、開始導線
- `buildModeCosts.ts` ビルドモードの直接指定 OP コスト定義
- 観測モードでは観測テーマだけを選び、所属環境・体格・取口・成長・素質は完全ランダムに任せる
- ビルドモードでは体格、取り口、成長型、付出・入門資格、素質、所属環境、リスク傾向など、直接能力値ではない前提をまとめて選ぶ。良い前提ほど OP コストを高くする
- 体格の「標準型」は「おまかせ」とは別扱いで、極端な小兵・大型を避けて平均体格へ寄せる軽い有料指定とする

## 設計ルール

- bias の定義と合成は `src/logic/archive/` に置く
- ビルドモードの直接指定コストは `buildModeCosts.ts` で管理し、観測テーマ / modifier コストと合算して支払う
- 体格・リスク傾向などの modifier は、独立した「読み口の調整」欄ではなくビルド設定の各軸として表示する
- 所属環境は 45 部屋を直接選ばせず、`src/logic/simulation/heya/` の部屋系統から既存 `stableId` へ解決する
- UI は上部 HUD、中央の選択ボード、右側の新弟子札、下部確認バーで構成し、フォームではなく選定作業として見えるようにする
- 新弟子札プレビューは選択した前提と読み味だけを表示し、8 軸能力値や潜在能力の数値は表示しない
- 画面は選択状態と説明表示だけを担当し、結果保証のような文言を出さない
- ビルドモードでも 8 軸能力値や潜在能力の数値入力は露出しない
- 本編 scout と同じ初期生成導線を使い、観測ビルド専用の別モデルを作らない
